# 振り返りレポート (KPT)

## 評価の切り口 (4 Axes)
1. **安全性 (Safety)**
2. **効率性 (Efficiency)**
3. **コンテキスト (Context)**
4. **合意形成 (Alignment)**

## 1. K (Keep): 継続すべき成功パターン
- **[Context]:**
  - Copilotによるレビューが、些細だが重要な整合性の欠如（略語の不統一）を検出した。人間が見落としがちな記述の揺れを機械的に指摘するプロセスが機能している。

## 2. P (Problem): 構造的な阻害要因
- **[Context]:**
  - `gemini:tdd` ラベルの説明文が `(Impl)` となっており、スキル名 `implementing-python-tdd` や他のラベル `(Arch)`, `(Spec)` との整合性が取れていなかった。
  - テンプレート作成時に、用語の定義（TDD vs Impl）が揺らいでいた、あるいはコピー元の記述が古かった可能性がある。

## 3. T (Try): 具体的な改善策と資産化
- **立案された仮説:**
  - **実証的:** 指摘通り `(TDD)` に修正する。
  - **飛躍的:** ラベルの定義と説明文を管理する単一の定義ファイル（SSOT）を持ち、テンプレート生成時にそこから参照するか、Lintで検証できるようにする。
- **選択された改善策:**
  - 現時点では実証的な修正（テンプレートの修正）を行う。
  - 将来的には `metadata-logic-spec.md` 等でラベル定義を厳格化することを検討する。
