# 最終監査チェックリスト (Final Audit Checklist)

## 1. トレーサビリティ (Traceability)
- [ ] **意図との整合:** 設定された目標は、`analyzing-intent` で特定された「ユーザーの真の意図」を解決するものか？
- [ ] **事実の裏付け:** アクションプランは、`scouting-facts` で収集された「事実（コード、ドキュメント）」に基づいているか？（推測でファイルパスなどを決めていないか？）

## 2. 実行可能性 (Feasibility & Specificity)
- [ ] **コンテキスト充足:** その作業を行うために必要な情報は全て手元にあるか？（ファイルの中身を読み込んでいるか？）
- [ ] **検証の具体性:** 成功/失敗を判定するコマンドは、コピペで実行可能なレベルまで具体的か？
- [ ] **自律完結性:** ユーザーへの追加質問なしで、エージェント単独で完遂できるタスクか？

## 3. 安全性 (Safety & Constraints)
- [ ] **破壊的変更の回避:** 削除や上書きを行う場合、バックアップや復元手順が考慮されているか？
- [ ] **スコープ外の保護:** 変更してはいけないファイル（SSOT等）や、影響を与えてはいけないモジュールへの干渉がないか？

## 4. 判定 (Decision)
- **判定:** [ GO / NO-GO ]
- **理由:** (NO-GOの場合は、どのステップに戻って何を修正すべきか記述)
