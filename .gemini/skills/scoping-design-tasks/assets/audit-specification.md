# 設計指針監査チェックリスト (For Specification)

## 1. 要求の網羅性 (Requirements Coverage)
- [ ] **意図の反映:** 分析された「ユーザーの真の意図」が仕様に落とし込まれているか？
- [ ] **エッジケース:** 正常系だけでなく、異常系や境界条件についての考慮があるか？

## 2. 実装可能性 (Implementability)
- [ ] **詳細度:** 開発者が迷わず実装できるレベル（データ型、API定義など）まで掘り下げられているか？
- [ ] **技術的裏付け:** 既存のコードベースやライブラリとの整合性が取れているか？

## 3. 曖昧さの排除 (Unambiguity)
- [ ] **用語統一:** ユビキタス言語が使用されているか？
- [ ] **決定事項:** 「TBD」や「検討中」が、実装ブロッカーにならない範囲に収まっているか？
