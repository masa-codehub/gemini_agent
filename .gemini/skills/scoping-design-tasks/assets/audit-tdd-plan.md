# 設計指針監査チェックリスト (For TDD Plan)

## 1. テスト戦略 (Test Strategy)
- [ ] **ピラミッド:** 単体テストと結合テストのバランスは適切か？
- [ ] **Mocking:** モック化する境界（外部API、DBなど）が明確か？

## 2. 構造設計 (Structural Design)
- [ ] **レイヤー配置:** クラスや関数が適切なレイヤー（Domain, UseCase, Infra）に配置されているか？
- [ ] **依存関係:** 循環参照や不正な依存（Domain -> Infra）がないか？

## 3. テストケース (Test Cases)
- [ ] **Red/Green:** 失敗するテスト（Red）を先に書く手順になっているか？
- [ ] **網羅性:** 主要なビジネスロジックがテストでカバーされる計画か？
